# Project Architecture & Documentation

This document serves as an index and explanation of the codebase structure, model architecture, and all associated files.

## 📁 Active Codebase

### `model.py`
The core Machine Learning and Data Engineering pipeline.
- **Purpose**: Reads historical match data, computes complex features (like Elo ratings, rolling Expected Goals, recent form), and trains the ensemble model.
- **Architecture**:
  - **Data Processing**: Loads Kaggle datasets (`results.csv`, `shootouts.csv`, `goalscorers.csv`, etc.). Filters out friendlies to ensure high-quality training signal.
  - **Feature Engineering**: Calculates a custom strength-adjusted Elo rating system that updates chronically over 25,000 matches.
  - **Modeling**: Uses an ensemble `VotingClassifier` consisting of `XGBoost` (Gradient Boosting), `RandomForest` (Bagging), and `LogisticRegression` (Linear).
- **Outputs**: Generates `football_model.pkl` (the trained model) and `encoders.pkl` (label encoders and feature lookup tables like current Elo ratings).

### `worldcup.py`
The Web Dashboard and Prediction Server.
- **Purpose**: Serves the user interface and exposes an API endpoint for live predictions.
- **Architecture**:
  - **Backend**: Built with Flask. Loads the `.pkl` files generated by `model.py` directly into memory.
  - **Frontend**: Contains HTML/CSS strings rendering a single-page application (SPA). Features dynamic tabs, animated progress bars, and AJAX-driven live predictions.
  - **Logic**: Simulates World Cup groups to predict standings using the loaded ML model.

### `.venv/`
The Python Virtual Environment directory.
- Contains all installed dependencies required to run the project locally (e.g., `scikit-learn`, `xgboost`, `flask`, `pandas`).

## 🗄️ Archive Directory

The `archive/` folder contains historical code from earlier iterations of the project, as well as the raw CSV datasets. They are stored here to keep the root directory clean while preserving data for retraining.

### Data Files (Used by `model.py`)
- **`results (2).csv`**: Historical international football match results.
- **`shootouts (2).csv`**: Historical shootout records.
- **`goalscorers (2).csv`**: Records of individual goalscorers.
- **`players_22.csv`**: FIFA 22 video game ratings used to inject team quality metrics.
- **`former_names (2).csv`**: Mapping file to handle countries that have changed names.
- **`football.db`**: An old SQLite database file from a previous architecture.

### Obsolete Code Scripts
- **`app.py`**: An older, simpler version of the prediction dashboard.
- **`check.py`**, **`cleaning.py`**, **`database cleaning.py`**: Various scripts used in the past to clean the CSV files and push them to the SQLite database.
- **`database.py`**: Old SQLAlchemy ORM definitions for interacting with `football.db`.
- **`simulation.py`**: An older script that attempted to simulate tournaments before the logic was ported cleanly into `worldcup.py`.

## 🤖 Model Details

The model predicts the outcome of a match between a Home and Away team as a probability distribution over 3 classes: **Home Win, Draw, Away Win**.

- **Why an Ensemble?** Football is highly stochastic. By combining three fundamentally different algorithms, the model becomes robust. XGBoost catches complex interactions, Random Forest prevents overfitting through random subspace sampling, and Logistic Regression acts as a stable linear baseline.
- **The Elo Impact**: The most important feature by far is the difference in computed Elo rating between the two teams. Our custom Elo system starts teams at 1500 and updates their rating based on the outcome of matches, with the size of the update scaled by the importance of the tournament (e.g., a World Cup win shifts Elo more than a Friendly).
